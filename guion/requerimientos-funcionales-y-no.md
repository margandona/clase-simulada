# 🧠 Explicación simple para recordar

---

## 1️⃣ Requerimientos (idea general)

**Puedes empezar así:**

> Un requerimiento es una descripción de algo que un sistema debe hacer o debe cumplir para funcionar correctamente.

**Luego puedes agregar:**

> Los requerimientos ayudan a definir cómo debe construirse el sistema para satisfacer las necesidades de los usuarios.

---

## 2️⃣ Requerimientos de usuario

**Forma simple de decirlo:**

> Los requerimientos de usuario describen lo que el usuario espera o necesita que el sistema haga.

- Son más generales
- Están expresados en lenguaje sencillo

**Ejemplo:**

> Un usuario quiere poder consultar su saldo en una aplicación bancaria.
> Ese sería un requerimiento de usuario.

---

## 3️⃣ Requerimientos de sistema

**Puedes decir:**

> Los requerimientos de sistema describen de forma más técnica cómo el sistema debe comportarse para cumplir las necesidades del usuario.

- Son más específicos

**Ejemplo:**

> El sistema debe permitir consultar el saldo en menos de 2 segundos y mostrar el resultado en la pantalla.

---

## 4️⃣ Requerimientos funcionales

**Forma clara de explicarlo:**

> Los requerimientos funcionales describen las acciones o funciones que el sistema debe realizar.
> En otras palabras: lo que el sistema hace.

**Ejemplos:**

- registrar usuarios
- enviar señales
- calcular rutas
- transmitir datos

**En tu clase puedes decir:**

> En el caso de NOVA, una función sería calcular la trayectoria de la nave.

---

## 5️⃣ Requerimientos no funcionales

**Frase útil:**

> Los requerimientos no funcionales describen las condiciones o características bajo las cuales el sistema debe funcionar.
> En otras palabras: cómo debe funcionar el sistema.

**Ejemplos:**

- seguridad
- rendimiento
- disponibilidad
- tiempo de respuesta
- energía disponible

**Ejemplo con NOVA:**

> La nave necesita energía suficiente para funcionar.
> Eso sería una condición.

---

## 🧩 Forma ultra simple de recordarlo

**Regla rápida:**

- Función → lo que el sistema hace
- Condición → lo que el sistema necesita

---

## 🚗 Ejemplo que nunca falla (auto)

**Función:** conducir**Condición:** tener combustible

> Sin combustible, el auto no puede realizar su función.

---

## 🎤 Cómo decirlo en la clase

> En ingeniería de software trabajamos con requerimientos, que son descripciones de lo que un sistema debe hacer o debe cumplir.
> Algunos requerimientos describen funciones, es decir, acciones que el sistema realiza.
> Otros describen condiciones necesarias, es decir, características que el sistema necesita para funcionar correctamente.

---

## ⭐ Truco mental para cuando estés nervioso

**Recuerda esta frase:**

> Hace vs Necesita
> Función → hace
> Condición → necesita
