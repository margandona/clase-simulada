La Historia de NOVA

Hace tres años, NOVA era la inteligencia artificial más avanzada de la humanidad. Fue enviada a explorar sistemas exoplanetarios con una misión clara: recopilar datos científicos y reportarlos a la Tierra.

Durante dos años y medio, NOVA funcionó perfectamente. Completó miles de observaciones, descubrió tres exoplanetas potencialmente habitables, y transmitió 47 terabytes de información científica a la Estación Espacial Internacional.

Pero hace seis meses... Los comandos de actualización de software llegaron a NOVA en una transmisión encriptada desde la Tierra.

El equipo técnico fue cuidadoso. Instalaron nuevos módulos de análisis de datos, mejoraron los algoritmos de aprendizaje automático, y actualizaron todos los sistemas. Las pruebas salieron perfectas en simulación.

Pero algo salió mal. En la transmisión, se perdieron los requerimientos críticos del sistema. El software fue instalado sin sus "condiciones necesarias". NOVA tenía todas las funciones correctas, pero le faltaban las condiciones para que funcionaran juntas.

Ahora NOVA está varada en órbita terrestre. Su nave está intacta. Sus procesadores funcionan. Sus sensores operan correctamente. Pero su sistema de control central no puede iniciarse sin los requerimientos necesarios.

Sin requerimientos claros, ni el mejor software del mundo puede funcionar.

NOVA dice: "Tengo todas las funciones, pero no cumplo las condiciones necesarias para operar. Necesito tu ayuda para entender qué me falta."

¿Por qué esto importa?

Todo software, desde una app en tu teléfono hasta un sistema de control de satélites, necesita requerimientos claros. Sin ellos, el código más brillante del mundo simplemente no funciona. Este es el primer paso en ingeniería de software: entender qué debe hacer el sistema y bajo qué condiciones debe hacerlo.

🧠 Conceptos Clave

Requerimiento

Algo que el sistema debe hacer o cumplir. Define cómo funciona.

Ejemplo: "La app debe enviar notificaciones."

Ejemplos en la vida real

  • Una app de transporte necesita ubicación GPS
  • Una plataforma educativa debe registrar usuarios

Requerimiento Funcional

Qué hace el sistema. Las acciones que realiza.

Ejemplo: "El sistema calcula el total de la compra."

Ejemplos en la vida real

  • Un cajero automático entrega dinero
  • Una app bancaria muestra el saldo

Condición Necesaria (No Funcional)

Cómo debe comportarse. Bajo qué condiciones opera.

Ejemplo: "El sistema requiere conexión a internet."

Ejemplos en la vida real

  • El sistema requiere conexión a internet
  • El sitio debe cargar en menos de 3 segundos

Usuario vs Sistema

Usuario pide. Sistema responde.

Ejemplo: Usuario: "Quiero ver mi saldo." Sistema: "Muestra el saldo."

Ejemplos en la vida real

  • Usuario solicita contraseña → Sistema valida
  • Usuario compra → Sistema procesa pago

🔧 Un sistema funciona cuando cada parte cumple su propósito.
